# 序
Kyle Simpson 是一个彻底的实用主义者。

我想不出比这个更高的评价了。对于我来说，软件开发工程师必须要有两种重要的品质。是的，是必须，而不是应该。Kyle 对 JavaScript 编程语言敏锐的梳理能力，以及能够深入浅出地将它们呈现出来是首屈一指的。

You Don't Know JS 系列的读者可能对于 ES6 & Beyond 比较熟悉：他们希望能够沉浸在所有揭示那些理所当然的或者从来没有考虑过的语义的过程中，从显而易见到明锐感知。直到现在，You Don't Know JS book 系列至少对于那些熟悉它的读者来说涵盖了大部分知识。关于这些主题他们已经见过或者听过；他们甚至可能已经体验过。这本书封面涵盖的知识点只有一小部分开发者接触过：引入 2015 年 ECMAScript 语言规范。

在过去的几年中，我目睹了 Kyle 的在他少数专业的同行的帮助下通过不懈努力熟悉这门语言达到专业的水平。非常不错，当时考虑撰写本文时，语言规范的文档还没有正式出版！但是我说的是真的，我读了 Kyle 所写书的每一个字。我自始至终跟随文档的改变，内容越来越好，并且提供更深层次的理解。

这本书在新的知识点或者旧的知识点上都会动摇你的理解。目的是通过授予你新的能力，运用你的工具同步发展你的知识。它的存在是给你拥抱要下一代的 JavaScript 编程的信心。
# 前言
# 第一章：ES？ 现在和未来

在你深入本书之前，你应该对之前版本（在撰写本文时）的 JavaScript， 通常叫做 ES5 有一个扎实的基础。这里我们打算直接谈论即将到来的 ES6，以及超越现在，来展望 JS 将如何演变前进。

如果你仍然在寻找在 JavaScript 方面的信心，我强烈推荐你阅读本系列中的其他书籍：

- up & Going： 你是一个新的 Javascript 编程开发者吗？这是一张路线图作为你开始编程学习旅程的开始。
- 作用域 & 闭包：你知道 JS 的词法作用域是基于编译器语义（而不是翻译）？你能解释下闭包如何成为词法作用域和作为值的函数的直接结果吗？
- this & 对象原型： 你能背诵 this 的四种简单的绑定规则吗？你还在用假的”类“吗，而不是采用简单的”行为委托模式“？听说过”对象关联“吗？
- 类型 & 语法：你知道 JS 中的内置对象类型吗？你知道如何正确地强制类型转换吗？你对 JS 语法和词法的细微差别掌握如何？
- 异步 & 性能：你任然在用回调函数来处理异步问题吗？能够解释下 Promise 是什么，为什么/怎样解决“回调地狱”？你知道如何使用生成器来改善异步代码的易读性吗？什么才是成熟的 JS 程序优化和个人操作？

如果你已经读了上面那些书籍，并且对于那些已经涵盖的知识点烂熟于心，是时候我们深入到 JS 的演变，探索即将到来变化了。

不想 ES5，ES6 不是一组有节制的新的 API 的集合。它包含了许多新的语法形式，其中一些需要相当长的时间来适应。还有各种各样新的组织形式，和针对不同数据类型的 API 的补充。

ES6 是这门语言的一个激进的版本。即使你认为你知道 ES5，ES6 仍然是一个你不知道的新的东西。因此准备好，这本书探讨了所有 ES6 主要的主题，你需要跟上节奏。甚至让你一睹未来的特性，你应该意识到在未来 JavaScript 的轨迹。

警告：本书的所有代码都是 ES6+ 的环境下。在写这本书的时候，在浏览器和 JS 环境（比如： NodeJs）支持 相当多的 ES6 特性，因此你的情况可能有些不同。
